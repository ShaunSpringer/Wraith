    Overview

    The core Wraith View responsible for rendering a single instance of a view. It will bind to a models update event and re-render each time it changes. Additionally it will handle setting classes up based on the data-class directive, and other bindings that might be required in the future.

    - (void) bindClasses($view, model) (bound)

    Iterates over each of $view's children as well as $view itself and calls Wraith.BaseView#bindClasses to apply the class(es) accordingly. This requires the $view and its children to be marked up with data-class attributes.

    Parameters:

    • $view (HTMLElement) — The view to be binding the classes to/from
    • model (Wraith.Model) — The model to use when applying the classes

    - (void) bindClass($view, model) (bound)

    Binds a views data-class attribute -- which is represented as a set of colon delimited key-value pairs which are in turn separated by spaces, e.g. class1:value1 class2:value2 It takes the current value of the model given by dot notation (e.g. value1, value2 above) and casts the value as a boolean. If it is true the class (e.g. class1, class2) are applied during the rendering process.

    Parameters:

    • $view (HTMLElement) — The view to be binding the classes to/from
    • model (Wraith.Model) — The model to use when applying the classes
